RTL (Right-to-Left) Guidelines

Design principles and implementation guidelines for Arabic language support.

Core RTL Principles
↔️
Text Direction
All text flows from right to left in Arabic. Use CSS direction: rtl; on parent containers and components.
🔄
Layout Mirroring
Navigation sidebars, buttons, and icons should mirror positions. Use flexbox row-reverse or logical properties.
🎯
Visual Hierarchy
Important elements appear on the right. Reading patterns start from right side of interfaces.
📝
Numbers & Latin Text
Preserve left-to-right for numbers, dates, emails. Use unicode-bidi: embed; for mixed content.
🎨
Icons & Imagery
Mirror directional icons (arrows, chevrons). Keep symbolic icons unchanged if culturally neutral.

Accessibility Guidelines

Standards for creating an inclusive design that works for all users.

WCAG 2.1 Compliance
🔤
Color Contrast
All text meets WCAG AA contrast ratio of 4.5:1 for normal text, 3:1 for large text.
⌨️
Keyboard Navigation
All interactive elements accessible via Tab key. Visible focus indicators on all states.
🎤
Screen Reader Support
Semantic HTML, ARIA labels, proper heading hierarchy, alt text for images.
🎯
Touch Targets
Minimum 44x44px click/touch targets for mobile accessibility.
⏱️
Motion & Animation
Respect prefers-reduced-motion. Avoid automatic animations lasting over 5 seconds.

Form Patterns

Standard patterns for common form interactions and validations.

Form Best Practices
🏷️
Labels
Always use explicit labels associated with inputs. Position above on mobile, beside on desktop.
✓
Validation
Provide real-time validation feedback. Clear error messages indicating what and how to fix.
💾
Progress Indication
Show progress for multi-step forms. Allow saving and resuming incomplete forms.
⌛
Loading States
Provide feedback during form submission. Disable submit button to prevent duplicates.

Data Visualization Standards
